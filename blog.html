Eighteen out of Twenty
11/28/15 12:02 AM
My computer science professor used the phrase "eighteen out of twenty" to describe why Java's syntax for Array length and String length is different (.length vs. .length()). The developers of Java made a small mistake; two groups had created different conventions. But they were tired, and getting an 18/20 on the "assignment" of making Java was good enough. I think my professor was trying to describe it in terms that a college student would understand: the makers of Java felt it was good enough and were too tired to put in the extra effort for a mere two points.

Personally, I agonize over missing 2 points on an assignment. But honestly, with this website, I'd have to say I'm okay with 18/20. It's actually probably closer to 14/20, because of internal issues that I will explain further down. Now if I was a successful, rich, popular developer, I would aim for 20/20. But perfection isn't important just yet because this is a learning experience more than anything else. Rather than try to get everything perfect the first time, I'm going to focus on improving my abilities, which in turn will organically invoke improvement on my actual projects.

So what needs to be improved? Take a look at the source files for this site (the CSS file too). They are unorganized, cluttered, redundant, and inconsistent. AND, my blogging system is just adding more text to the blog.html webpage. This blog post was literally typed into the HTML file. It's not terrible I suppose, but what if I changed the template? What if I redesigned the HTML containing the blog posts? I would have to go back through each post and change it. Same with the navigation bar, the header, the footer, and the meta data on each page. Typing everything by hand only works until you need to change something.

The solution is to write a program that takes an HTML template file and several text source files, and generates the website. I think this is what Jekyll does. I could learn how to use Jekyll, but it will be more fun and more educational if I write the program myself. Rather than limiting myself to knowing how to generate websites with a specific program, I can unravel the concepts myself, ensuring that I will be able to do the same thing 10 years from now when Jekyll is outdated.

Tomorrow I need to return to homework, research and studying, so for now, typing the blogs directly into the HTML file is going to have to be acceptable.
Getting a Playable Game Online
11/27/15 12:05 AM
I made tons of progress with the website. It's still fairly simplistic, but I'm happy with the overall appearance of it, especially the Super Smash Pong page because I was able to upload the game, and it is currently playable on the website. How awesome is that?!? I was very excited to see an actual game that is playable on a live website that I made. That is something I've never done before. I had to tweak the build settings and play around with some HTML and JS that I didn't fully understand, but the game fits nicely into the webpage.

I tried uploading FBLA Antivirus as well, since it's also a Unity game, but I didn't build it correctly. It's a long story, but basically I built it while in my dorm, took the USB drive to my cousin's house, and tried adding it to my website there. However the viewport is all messed up. So, I need to return to my laptop and build the game with the correct settings. But I left my laptop in my dorm, so I can't fix this until tomorrow evening, when I return.

My next goal is to get all my games onto the website, including the downloadable versions. In theory this shouldn't be too difficult. All I need to do is add the files to the repository and include links and screenshots on the games' respective pages. After that, my next task will be to write a Java program that generates my HTML pages using templates and source files. This will give me greater flexibility and make it less of a hassle when I decide I want to redesign this website (let's be honest, it looks awful right now).
The Website Is Made (Sort Of)!

Thanksgiving break gave me the edge I needed to finally overcome my homework deficit and get started on this website! Yesterday I built the layout of the website from scratch using HTML and CSS. That took a lot of work, because this is my first time working with CSS and I don't quite understand some of its features.
